# Guidance for Using the Official Tailwind CSS Documentation

During the development of this project, the developer must use the official **Tailwind CSS** documentation as the main source of technical reference.

The official documentation should be used to validate utility classes, responsive design patterns, layout strategies, spacing, typography, colors, customization, configuration, dark mode, plugins, optimization, and other Tailwind CSS-related resources.

## Recommendations for the Developer

- Consult the official Tailwind CSS documentation whenever there are questions about styling implementation.
- Prioritize the best practices recommended by the official Tailwind CSS documentation.
- Avoid implementing styling solutions based only on random examples found on the internet.
- Check whether the utility classes and configuration options being used are compatible with the Tailwind CSS version adopted in the project.
- Use the documentation as a reference for technical decisions related to UI styling, responsiveness, design consistency, and maintainability.
- Validate custom configuration changes carefully before applying them across the project.

## Objective

Ensure that the project is developed following best practices, updated standards, and officially supported Tailwind CSS resources.

[Consult the official Tailwind CSS documentation during the development of the project:](https://v2.tailwindcss.com/docs)
