# Guidance for Using the Official Next.js Documentation

During the development of this project, the developer must use the official **Next.js** documentation as the main source of technical reference.

The official documentation should be used to validate implementation patterns, route structure, project configuration, rendering, components, APIs, optimizations, deployment, and other framework-related resources.

## Recommendations for the Developer

- Consult the official documentation whenever there are questions about Next.js features.
- Prioritize the best practices recommended by the framework’s official team.
- Avoid implementing solutions based only on random examples found on the internet.
- Check whether the resources being used are compatible with the current version of the project.
- Use the documentation as a reference for technical decisions during development.

## Objective

Ensure that the project is developed following best practices, updated standards, and officially supported Next.js resources.

[Consult this documentation during the development of the project:](https://nextjs.org/docs)
