# Guidance for Using the Official TypeScript Documentation

During the development of this project, the developer must use the official **TypeScript** documentation as the main source of technical reference.

The official documentation should be used to validate type definitions, interfaces, type aliases, generics, utility types, enums, modules, configuration, strict mode, compiler options, JavaScript interoperability, and other TypeScript-related resources.

## Recommendations for the Developer

- Consult the official TypeScript documentation whenever there are questions about typing, configuration, or language features.
- Prioritize the best practices recommended by the official TypeScript documentation.
- Avoid implementing type solutions based only on random examples found on the internet.
- Check whether the features being used are compatible with the TypeScript version adopted in the project.
- Use the documentation as a reference for technical decisions related to type safety, maintainability, and code quality.
- Validate compiler configuration changes carefully before applying them across the project.

## Objective

Ensure that the project is developed following best practices, updated standards, and officially supported TypeScript resources.

[Consult the official TypeScript documentation during the development of the project:](https://www.typescriptlang.org/pt/docs/)
