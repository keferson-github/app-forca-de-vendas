# Guidance for Using the Official Prisma Documentation

During the development of this project, the developer must use the official **Prisma** documentation as the main source of technical reference.

The official documentation should be used to validate database modeling, schema configuration, migrations, Prisma Client usage, queries, relations, seeding, environment variables, deployment practices, and other Prisma-related resources.

## Recommendations for the Developer

- Consult the official Prisma documentation whenever there are questions about database implementation.
- Prioritize the best practices recommended by the official Prisma team.
- Avoid implementing database solutions based only on random examples found on the internet.
- Check whether the resources being used are compatible with the current version of Prisma used in the project.
- Use the documentation as a reference for technical decisions related to the database layer and ORM configuration.
- Validate migration strategies carefully before applying changes to production environments.

## Objective

Ensure that the project is developed following best practices, updated standards, and officially supported Prisma resources.

[Consult the official Prisma documentation during the development of the project:](https://www.prisma.io/docs)
